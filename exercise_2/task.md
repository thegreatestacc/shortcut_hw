# Практическое задание: генератор задач и пул воркеров

## Задача

Реализуй систему:

1. Есть генератор задач (main), который создаёт N "заданий" (например, числа от 1 до N).
2. Есть канал jobs для заданий.
3. Есть M воркеров (горутин), каждая читает из jobs, обрабатывает (например, возводит число в квадрат с time.Sleep), пишет результат в канал results.

---

## Условия

1. Используй буферизованный канал jobs (ёмкость выбрать разумно).
2. После отправки всех задач jobs закрывается.
3. Отдельная горутина читает из results и печатает результаты.
4. WaitGroup используется, чтобы дождаться окончания всех воркеров и закрыть results.

---
## Расширения
1. Сделай версию, где results не буферизован, и сравни поведение (блокировки/порядок вывода).
2. Добавь ограничение на одновременное количество "тяжёлых" задач через дополнительный семафор‑канал.
